Backend¶

The contract a backend must respect:

class ragger.backend.BackendInterface(firmware: Firmware)¶

Initializes the Backend

Parameters:

firmware (Firmware) – Which Firmware will be managed

abstract both_click() → None¶

Performs a click on both buttons of the device.

This method may be left void on backends connecting to physical devices, where a physical interaction must be performed instead. This will prevent the instrumentation to fail (the void method won’t raise NotImplementedError), but the instrumentation flow will probably get stuck (on further call to receive for instance) until the expected action is performed on the device.

Returns:

None

NoneType

exchange(cla: int, ins: int, p1: int = 0, p2: int = 0, data: bytes = b'') → RAPDU¶

Formats and sends an APDU to the backend, then receives its response.

The length is automaticaly added to the APDU message.

Parameters:

  • cla (int) – The application ID

  • ins (int) – The command ID

  • p1 (int) – First instruction parameter, defaults to 0

  • p1 – Second instruction parameter, defaults to 0

  • data (bytes) – Command data

Raises:

ExceptionRAPDU – If the raises attribute is True, this method will raise if the backend returns a status code not registered as a valid_statuses

Returns:

The APDU response

Return type:

RAPDU

exchange_async(cla: int, ins: int, p1: int = 0, p2: int = 0, data: bytes = b'') → Generator[None, None, None]¶

Formats and sends an APDU to the backend, then gives the control back to the caller.

This can be useful to still control a device when it has not responded yet, for instance when a UI validation is requested.

The context manager does not yield anything. Interaction through the backend instance is still possible.

Parameters:

  • cla (int) – The application ID

  • ins (int) – The command ID

  • p1 (int) – First instruction parameter, defaults to 0

  • p1 – Second instruction parameter, defaults to 0

  • data (bytes) – Command data

Raises:

ExceptionRAPDU – If the raises attribute is True, this method will raise if the backend returns a status code not registered as a valid_statuses

Returns:

None

Return type:

NoneType

Responses¶

Most APDU response are instances of class RAPDU:

class ragger.RAPDU(status: int, data: bytes)¶

However, if the backend has its raise_policy configured to raise on the received APDU response, it will raises an ExceptionRAPDU:

class ragger.ExceptionRAPDU(status: int, data: bytes = b'')¶

The different values of RaisePolicy are :

class ragger.backend.RaisePolicy(value)¶

An enumeration.
